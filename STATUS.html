the shape

One folder = one design system

No studio, no multi-project orchestrator, no client copies. A single package (@dsk/core) holds the design-system store at its center — the living memory — and three faces sit over it. The store is the source of truth; everything else is a projection of it.

◆ The store — the design system itself
A human-readable markdown file repo (dsk init --files) or SQLite. Tokens, components + their usage rules, patterns, guidelines, IA, flows, docs, assets.
face 1 · builder

The dsk CLI

The machine that helps you build the system: set tokens, define components with usage rules, add patterns/guidelines, import, export.

$ dsk token set · component · export
face 2 · viewer

dsk serve

A React app that boots straight into this one design system: token gallery, atomic component browser, IA + user-flow canvas, docs, assets.

$ dsk serve → localhost:4321
face 3 · agent

MCP + exports

The AI's live connection (10 MCP tools) + a static export bundle, so Claude Code can look things up, get its plan checked, and write new work back.

.mcp.json · design.md · AGENTS.md

honesty check

What the audit caught

You asked me not to claim anything that isn't verifiably working. Running the system for real turned up one genuine defect — reported here rather than hidden.

🐛 → ✅ Fixed: dsk pattern / dsk guideline crashed on a file-repo project

When an optional field was omitted, the CLI passed undefined into the markdown front-matter, and the YAML dumper throws on undefined. The SQLite backend was unaffected (it serializes through JSON), so it had gone unnoticed — but the file repo is the default storage. Fixed by normalizing the data the same way SQLite does before writing, and locked in with a regression test (test count 26 → 27).

Run it yourself

The exact path used in this audit. From the repo root.

# health
npm run typecheck        # 5 packages, clean
npm test                 # 27 passing

# build a design system (human-readable file repo)
mkdir acme && cd acme
npx dsk init --files --name "Acme"
npx dsk import ../examples/seed.acme.json
npx dsk recommend "confirm a destructive action"   # → ConfirmDialog
npx dsk export           # → tokens.css · design.md · AGENTS.md · SKILL.md

# view it + serve the agent's MCP (one process)
npx dsk serve            # → localhost:4321  (visualizer + /mcp)
